# UI/UX ресёрч топов — что я реально видел (2026-07-04)

> Скриншоты — в `ui-refs/` рядом (сняты мной с лендингов/README продуктов). Коллаж: `ui-refs/ui-refs-collage.jpg`.

## ZCode (z.ai) — `zcode-hero.jpeg`, `zcode-2.jpeg`
- **Компоновка:** слева сайдбар (New Task ⌘N / Open Workspace / Skills + список задач с таймстампами по workspace), **центр — тред задачи**, **справа панель: Git tools (Changes +734 −7, ветка, Commit) + Goal (цель, 5/5 шагов, 2m, 89K tokens) + Progress (чеклист с зачёркиванием сделанного)**.
- Тул-события инлайн в треде, компактно: «Ran node --check app.js», «Updated styles.css +1 −7».
- Сверху треда — чипы: workspace и **ветка**.
- Тёмный, глубокий чёрный, почти монохром; UI-шрифт гуманистический + моно для кода/команд.
- Вывод: правая контекст-панель с git-фактами и прогрессом агента — сильнейший паттерн ZCode.

## Orca (onorca.dev) — `orca-hero.jpeg`, `orca-review.jpeg`, `orca-git.jpeg`
- **Компоновка:** слева Projects → задачи (подпись = ветка, цветная статус-точка: янтарная working / зелёная ready / серая idle; у активной бейдж `primary`), **центр — ВКЛАДКИ агент-сессий** (как вкладки браузера), внутри — терминальный лог агента («Working (esc to interrupt)», модель+cwd в статус-строке), справа — файловое дерево.
- **Review AI output = встроенный браузер** localhost:3000: «Click an element to add feedback for the agent» → «Notes sent to Claude Code». Визуальный фидбек по живому приложению, не по коду.
- **Статусбар: живые лимиты подписки** — «58% 5h · 41% wk» на провайдера. Killer-деталь.
- Мобильный компаньон: PINNED/ACTIVE задачи, «awaiting permission · sudo apt install» как ожидание ввода.
- Инлайн-дифф в треде (красный/зелёный, ±строки).
- Вывод: терминало-центричный IDE-вайб; фидбек-«аннотации» у Orca — про живой браузер, а не про дифф (дифф-комменты — у нас останется своей фичей).

## Conductor (conductor.build) — `conductor-hero.jpeg`
- **Компоновка:** слева воркспейсы по репо, карточки задач: **бейдж диффа (+312 −332), статус «Ready to merge» (зел.) / «Merge conflicts» (красн.), хоткей ⌘1…⌘9 на каждую задачу**. Центр — тред с **вкладками-тредами внутри задачи** (All changes / Debugging… / Review branch changes). Справа — панель Changes: **зелёный PR-бар «PR #1432 · Ready to merge · [Merge]»** + файлы с ±.
- Весь UI моноширинный — жёсткий терминальный характер (на любителя).
- Юзер-сообщения в треде — выделенные цветные блоки (легко сканировать диалог).
- Вывод: статусы готовности к мёржу прямо на карточках + мгновенное ⌘-переключение задач — лучший «командный пункт» из виденного.

## vibe-kanban (BloopAI) — `vibekanban-board.jpeg`, `vibekanban-workspace.jpeg`
- **Единственный настоящий канбан**: колонки To do / In progress / In review / Done; карточка показывает живой прогресс агента («just now», спиннер). **Задача открывается ДРОВЕРОМ поверх доски** (справа), не отдельным экраном: тред + композер с выбором модели + **Queue** (сообщения в очередь, пока агент работает) + Stop + «Open Workspace».
- Workspace-вью (тёмный): слева список воркспейсов с ±бейджами, центр — **структурированный отчёт агента** (Summary / Root Cause / The Fix / How It Works Now) + бейдж «2 files changed +61 −3» над композером, справа — side-by-side дифф + Git-панель (ветка, **Open pull request**), Terminal, Notes.
- Светлая тема доски выглядит наименее цельно из всех продуктов.
- Вывод: Queue-композер и структурированный финальный отчёт агента — паттерны, которые стоит взять; канбан как основной вид — нет.

## Linear (linear.app) — `linear.jpeg`
- Показывают агентские фичи через **активити-фид задачи**: «Linear moved from Todo to In Progress · just now», «Changed 2 files · Draft PR awaiting your review», карточка draft-PR (master ← ветка) с Preview; композер «Tell Linear what to do next…».
- Полировка, плотность, клавиатурность — эталон качества исполнения.
- Вывод: события агента как лента фактов в задаче = наш журнал, поднятый в UI.

## Zed (zed.dev) — `zed-agent.jpeg`
- Агент-панель в редакторе, справа список прошлых тредов с таймстампами. Минимализм, «does its job and stays standby until the next time» (их же цитата). Монохром + один синий акцент.

## Сквозные выводы для gcode
1. **Индустрия сошлась на IDE-компоновке, не на «доске-экране»:** ZCode/Orca/Conductor — везде слева постоянный список задач, в центре тред выбранной, справа контекст (git/файлы/прогресс). Канбан только у vibe-kanban, и тот открывает задачу дровером, не уводя с доски. Никто не делает «отдельный экран доски → отдельный экран задачи» с навигацией туда-сюда.
2. **Правая контекст-панель — стандарт:** git-факты (ветка, ±, changes), у ZCode ещё Goal/Progress агента. Наш B1 подтверждён, причём панель должна быть **per-repo** (наш мультирепо — уникум, у всех остальных задача = один репо!).
3. **Статус готовности к git-действию на карточке** (Conductor «Ready to merge»/«Merge conflicts») — для нас идеально ложится на ops-из-фактов.
4. **Хоткеи на задачи (⌘1-9)** — Conductor; мгновенное переключение без мыши.
5. **Композер с Queue** (vibe-kanban) — очередь сообщений при работающем агенте: ровно наше решение, подтверждено.
6. **Лимиты подписки в статусбаре** (Orca, % 5h/weekly) — маленькая деталь с огромной пользой.
7. **Структурированный финальный отчёт агента** (vibe-kanban Summary/Fix) + **активити-лента фактов** (Linear) — «анти-хаос» в чистом виде.
8. **Тёмная тема — везде** (кроме доски vibe-kanban, и она смотрится слабее всех). Моно-шрифт как ХАРАКТЕР (Conductor) — на любителя; ZCode-баланс (гуманистический UI + моно-акценты) приятнее.
9. Чего НЕТ ни у кого из виденных: мультирепо-задача (везде задача = 1 репо/1 worktree), группы с integration-веткой, ops-слой из фактов CI. Наши уникумы остаются уникумами.
