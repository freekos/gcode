# Фаза 3a — UI/UX дизайн 📖

> Это дизайн-документ: ни строчки кода, пока мы вместе не зафиксируем, как gcode выглядит и ощущается. Дальше 3b строит ровно то, что здесь утверждено.

## Принципы (мерило любого решения ниже)
1. **Анти-хаос:** в любой момент видно — что ИИ сделал / делает / собирается делать. Ничего не происходит «где-то там».
2. **Клавиатура — первична** (референс Linear): всё достижимо без мыши, мышь — удобное дополнение.
3. **Скорость = вежливость:** отклик мгновенный, анимации быстрые и незаметные; ничего не «грузится» у тебя на глазах без индикации.
4. **Работа — в задачах, не в приложении:** gcode не просит внимания, он его экономит. Открыл → понял состояние за 3 секунды → пошёл в нужную задачу.

## Карта приложения (что где живёт)
```
┌────────────────────────────────────────────────────────────┐
│  Главное окно                                              │
│  ┌──────────┐  ┌───────────────────────────────────────┐   │
│  │ Сайдбар  │  │  Центр: список задач проекта          │   │
│  │ проекты  │  │  (или экран задачи, если открыта)     │   │
│  └──────────┘  └───────────────────────────────────────┘   │
│                                                            │
│  Поверх всего (глобальные, из любого места):               │
│   • Cmd+K  — командная палитра (всё: задачи/действия)     │
│   • Cmd+J  — AI-ассистент (что происходит? что вчера?)    │
│   • Cmd+O  — ops-модалка (фаза 7)                          │
└────────────────────────────────────────────────────────────┘
```
- **Один уровень навигации:** проект → задача. Никаких вкладок-в-вкладках.
- **Модалки глобальные с историей** (твоё решение из cc): открываются где угодно, помнят прошлое.

## Главный экран — развилка A (выбери)

**Вариант A1 — список со статус-группами (Linear-стиль). Моя рекомендация.**
```
┌──────────┬─────────────────────────────────────────────┐
│ azi      │  ⏳ Требуют внимания (2)                    │
│ invictus │   ● fix-login      review   2 repos  5m ago │
│ gcode    │   ● price-bug      needs input      1h ago  │
│          │  ⚙ Работают (1)                             │
│ + проект │   ◐ add-tests      running  ▂▃▅ live        │
│          │  📋 Остальные (3)                           │
│          │   ○ refactor-api   new              вчера   │
└──────────┴─────────────────────────────────────────────┘
```
Почему: «вернулся через день» решается сортировкой — сверху то, что ждёт ТЕБЯ (review/needs_input), потом живое, потом остальное. Плюс это компактно при 20+ задачах.

**Вариант A2 — канбан-колонки (vibe-kanban-стиль).**
```
│  New       │ Running    │ Review     │ Done      │
│ ┌────────┐ │ ┌────────┐ │ ┌────────┐ │           │
│ │refactor│ │ │add-test│ │ │fix-login│ │  ...     │
│ └────────┘ │ └────────┘ │ └────────┘ │           │
```
Почему нет (моё мнение): колонки тратят ширину, при 3-5 задачах смотрятся пусто, при 30 — нечитаемы. Канбан можно добавить позже как переключаемый вид, если захочется.

## Экран задачи — развилка B (выбери)

**Вариант B1 — тред в центре, контекст справа. Моя рекомендация.**
```
┌──────────┬───────────────────────────────┬──────────────┐
│ ← назад  │  ТРЕД (чат с агентом)         │ Задача       │
│          │                               │ 2 worktrees: │
│ треды:   │  Ты: почини редирект…         │  server ✎3   │
│ #1 login │  ◐ Агент: смотрю код…         │  crm    ✎1   │
│ #2 tests │  [tool: Edit auth.ts]         │ ветка:       │
│          │  Агент: готово, изменил…      │  fix-login   │
│          │  ┌─────────────────────────┐  │ статус:      │
│          │  │ Написать сообщение…   ⏎ │  │  ● review    │
│          │  └─────────────────────────┘  │              │
└──────────┴───────────────────────────────┴──────────────┘
```
Правая панель = «что ИИ сделал»: живой счётчик изменённых файлов по репо (✎3), ветка, статус. В фазе 4 клик по ✎3 открывает дифф.

**Вариант B2 — контекст сверху полосой, тред на всю ширину.** Экономит правую панель, но «что сделал агент» уезжает из поля зрения — против принципа №1.

## Ключевые UX-решения (без развилок — скажи, если против)
- **Создание задачи = 2 действия:** `Cmd+N` → название (+Tab: выбрать репо, по умолчанию ВСЕ) → `Enter`. Провижининг идёт фоном, карточка сразу на доске со спиннером «готовлю worktrees…».
- **Агент работает** → карточка пульсирует, в треде живой стрим; поле ввода в треде при работающем агенте превращается в «⏸ Агент работает — сообщение отправится после» (очередь, не отказ; отказ — только на второй параллельный ЗАПУСК).
- **needs_input** — самый громкий статус: задача поднимается в топ, бейдж с вопросом агента прямо на карточке.
- **Пустые состояния учат:** пустая доска показывает «⌘N — первая задача», а не белый экран (анти-туман, discoverability).
- **Realtime без перезагрузок:** статусы/счётчики обновляются сами (ядро шлёт события в UI).

## Дизайн-язык
- **Тема:** тёмная — основная (мы живём в тёмных редакторах; референс глубины — ZCode, лаконичности — Zed). Светлая — после 1.0.
- **Шрифты:** UI — Inter; код/слаги/ветки — JetBrains Mono. Никаких третьих шрифтов.
- **Палитра статусов** (единственные яркие цвета в интерфейсе): new — серый, running — янтарный пульс, needs_input — оранжевый, review — синий, done — зелёный. Остальной UI — почти монохром, цвет = сигнал.
- **Движение:** 120–160ms, ease-out, только opacity/transform (это дёшево для GPU → плавно). Стрим текста — как в Claude: мягкое появление, без дёрганья.
- **Плотность:** компактная, как инструмент на каждый день (Linear, не лендинг). Скругления небольшие, тени едва заметные.
- Всё это — **дизайн-токены с первого дня** (цвета/отступы/шрифты/тайминги в одном файле): «нормальный UI» потом не переделывается, а настраивается.
