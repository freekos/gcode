# Фаза 3a — UI/UX дизайн 📖 (v2 — переписан после визуального ресёрча)

> Ни строчки кода, пока не зафиксируем дизайн вместе. **v2: я реально посмотрел интерфейсы ZCode, Orca, Conductor, vibe-kanban, Linear, Zed** — скриншоты в [`../research/ui-refs/`](../research/ui-refs/), разбор в [`../research/ui-research.md`](../research/ui-research.md), коллаж пришлю отдельно.

## Принципы (мерило любого решения)
1. **Анти-хаос:** видно, что ИИ сделал / делает / собирается делать — всегда, без раскопок.
2. **Клавиатура первична** (Linear, Conductor ⌘1-9): всё без мыши.
3. **Скорость = вежливость:** мгновенный отклик, анимации 120–160ms, ничего не «грузится» молча.
4. **gcode экономит внимание:** открыл → понял состояние за 3 секунды → пошёл в нужную задачу.

## Главный факт ресёрча
**ZCode, Orca и Conductor сошлись на одной компоновке** — и это не канбан и не «доска-экран»:
```
┌ Сайдбар ──────┬ Центр ─────────────────────┬ Контекст ────────┐
│ список задач  │ тред выбранной задачи      │ git-факты, файлы,│
│ со статусами  │ (агент стримит сюда)       │ прогресс агента  │
└───────────────┴────────────────────────────┴──────────────────┘
```
Задачи всегда на глазах слева, переключение мгновенное (Conductor: ⌘1…⌘9), никакой навигации «экран доски → экран задачи → назад». Канбан-доску делает только vibe-kanban — и даже он открывает задачу **дровером поверх доски**, чтобы не терять контекст (и его светлая доска выглядит слабее всех).

## Развилка A — компоновка приложения

**A1 — единый IDE-layout (как ZCode/Orca/Conductor). Моя рекомендация.**
```
┌──────────────┬──────────────────────────────┬───────────────┐
│ ПРОЕКТ ▾ azi │  fix-login  [ветка] [review] │ per-repo:     │
│              │ ──────────────────────────── │  server  ✎3   │
│ ⏳ внимание   │  Ты: почини редирект…        │  crm     ✎1   │
│ ●fix-login ⌘1│  ◐ Агент: смотрю код…        │ ─────────────│
│ ●price-bug ⌘2│  [tool: Edit auth.ts]        │ Goal 3/5 ✓✓✓  │
│ ⚙ работают   │  Агент: готово, изменил…     │ (прогресс     │
│ ◐add-tests ⌘3│                              │  агента)      │
│ 📋 остальные  │ ┌──────────────────────────┐ │               │
│ ○refactor  ⌘4│ │ Сообщение… (Queue если ⚙)│ │               │
│              │ └──────────────────────────┘ │               │
│ + задача ⌘N  │                              │               │
├──────────────┴──────────────────────────────┴───────────────┤
│ статусбар: ↯ 58% 5h · 41% wk   ● 2 агента работают          │
└──────────────────────────────────────────────────────────────┘
```
Сайдбар = мой прежний «список со статус-группами» (внимание → работают → остальные). Пустой центр (задача не выбрана) показывает сводку проекта и «⌘N — новая задача».

**A2 — канбан-доска + задача дровером (vibe-kanban).** Честно: из виденного этот путь выбрал один продукт из пяти, и это самый слабый визуально. Беру в план только как возможный будущий «вид» поверх тех же данных.

## Развилка B — правая контекст-панель задачи

**B1 — панель с git-фактами per-repo + прогресс агента (ZCode-паттерн, адаптированный под наш мультирепо). Моя рекомендация.**
- **Per-repo блок** (наш уникум — у всех топов задача = 1 репо, у нас N): на каждый репо — изменённые файлы (✎3, +61 −3), ветка. Клик → дифф (фаза 4).
- **Goal/Progress агента** (как ZCode): что агент делает сейчас, чеклист его плана, зачёркнутое сделанное. «Анти-хаос» вживую.
- **Статус готовности** (как Conductor): «чисто / есть незакоммиченное / готово к PR» — из git-фактов (фаза 5 добавит кнопки действий).

**B2 — тред во всю ширину, контекст сверху чипами (как у ZCode чипы workspace/ветка, но без панели).** Компактнее, но «что сделал агент» уходит с глаз — против принципа №1.

## UX-решения, подтверждённые ресёрчем (беру как есть)
- **Композер с Queue** (vibe-kanban): агент работает → сообщение встаёт в очередь с пометкой, не отказ. Стоп-кнопка рядом.
- **⌘1…⌘9** — мгновенное переключение задач (Conductor), **⌘N** — новая задача (везде), **⌘K** — палитра.
- **Лимиты подписки в статусбаре** (Orca): «58% 5h · 41% wk» — ты сам делал это в statusline, теперь оно живёт в продукте.
- **Активити-лента фактов в задаче** (Linear): «агент начал · изменил 2 файла · задача → review» — наш журнал ядра, поднятый в UI.
- **Структурированный финальный отчёт агента** (vibe-kanban): Summary / что изменил по репо — просим агента (CLAUDE.md правило уже есть) и рендерим красиво.
- **Тул-события инлайн одной строкой** (ZCode): «[Edit] auth.ts +4 −1» — не простыни JSON.
- Создание задачи: **⌘N → название (+Tab: репо, по умолчанию все) → Enter**; карточка сразу, provisioning фоном со спиннером.
- **needs_input — самый громкий статус:** топ сайдбара + бейдж с вопросом.
- Пустые состояния учат хоткеям (анти-туман).

## Дизайн-язык (проверен на живых референсах)
- **Тёмная тема** — основная (все сильные продукты тёмные; светлая доска vibe-kanban — худший вид из увиденного). Светлая — после 1.0.
- **Характер: «ZCode-баланс»** — гуманистический UI-шрифт (Inter) + моно для кода/веток/команд (JetBrains Mono). Полностью моноширинный Conductor — колоритно, но утомляет; полный отказ от моно — теряет «инженерность».
- **Цвет = только сигнал:** статусы (new серый / running янтарный пульс / needs_input оранжевый / review синий / done зелёный), диффы (+зел/−красн), один акцент. Остальное — монохром (Zed).
- **Движение:** 120–160ms ease-out, только opacity/transform. Стрим текста — мягкое появление.
- **Плотность:** компактная (Linear/Conductor), не «воздушный лендинг».
- Всё — **дизайн-токены с первого дня** (файл с цветами/отступами/таймингами): UI дальше растёт настройкой, не переделкой.
