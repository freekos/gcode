## 1. Ответ на твой главный вопрос (azi: с нуля vs существующий, планы, CLAUDE.md)

**Существующий azi** — работает уже сейчас: «+ Добавить проект» → папка `~/Codebase/work/azi` (4 репо). Задачи-исследования («разберись, как устроен бэк») агент делает в worktree, корневой `CLAUDE.md`/`PRODUCT.md` подхватывает автоматически.

**Azi с нуля** — честно: флоу сырой. Наш scan ждёт папку с git-репо; пустая папка «начни здесь» не даст создать задачу. Мастер новых проектов — это фаза Project knowledge.

**«Как видеть планы проекта (фазы) и CLAUDE.md?»** — сегодня НИКАК, кроме «Показать файлы» → открыть файл руками. Это дыра, и она — ровно **фаза 6 Project knowledge**: фазы-доки проекта как объект UI (как docs/plan у gcode — ты по нему живёшь и он тебе нравится как процесс), CLAUDE.md на виду, вики по репо (тот самый Repo wiki). **Твоя задача про azi («понять, разделить на части, начать с core») — это и есть фаза 6, а не 5.** Предлагаю поменять их местами.

## 2. Инвентарь: что построено и моя честная оценка

### Ядро (фазы 0-2) — ✅ фундамент, сомнений нет
SQLite-состояние, worktrees на все репо, ветка сразу, deny-git для агента (проверен вживую), архив с авто-патчем, один агент на задачу, статусы из фактов. 74 теста. Этим пользуется всё остальное.

### Создание задач + нейминг — ✅ подтверждено тобой в бою
Мгновенное создание, ИИ-имя+ветка догоняют, скелетоны.

### Чат (фаза 3b/4) — 🟡 работает, но НЕ закончен. Известные дыры:
- **очередь сообщений живёт в памяти окна** — перезапустил приложение → очередь пропала;
- **нет редактирования своего сообщения** (retry есть, edit нет);
- **[+] вложения в композере — заглушка** (картинки/файлы агенту не передать);
- **треды**: ядро умеет несколько тредов на задачу, UI показывает только последний — нет «нового треда» и переключателя;
- **лимиты**: живой rate-limit баннер не проверен на реальном лимите;
- markdown: таблицы/ссылки не рендерятся (заголовки/списки/код — да);
- нет поиска по треду.

### Правая панель — 🔴 тебе не нравится, согласен что сырая:
- Goal/PROGRESS обновляется только после хода агента, не live;
- «✎ N +A −D» — числа есть, но нет главного: **что за задача ветки, ahead/behind, что дальше делать**;
- «ещё N не тронуты» — свёрнуто и бесполезно;
- нет никаких действий (всё ждёт ops фазы 8);
- визуально — список плашек без иерархии.
**Нужно твоё слово: что ты ХОЧЕШЬ видеть справа, когда смотришь на задачу?** Мой кандидат: (1) Goal+прогресс крупно, (2) репо с ветками и диффстатами → клик в дифф, (3) следующее действие.

### Дифф-ревью + редактор + деревья + ⌘L (фаза 4) — 🟢 код готов, но тобой в бою не проверен
Оценку дадим после твоей реальной недели. Риск: понастроили много входов (⌘P, ✎, дерево, Show files) — возможно, часть лишняя. Смотрим на использование, лишнее выпилим.

### Кандидаты «непонятно, нужно ли» (мои подозрения на выпил/упрощение):
- «Показать файлы» у ПРОЕКТА (дерево рабочей копии) — ты сам сомневался; оставили ради веток-бейджей. Если за неделю не откроешь — убираем кнопку.
- Сортировка «по статусу/по созданию» в меню вида — возможно, никто не переключает.
- Тумблер архивных в шапке — редкое действие на видном месте.
