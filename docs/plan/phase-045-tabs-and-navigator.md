# Фаза 4.5 — Вкладки по центру + правая панель-навигатор 🔨

По ревизии checkpoint-01 и решениям Газика. Сортировка и «Показать файлы» ОСТАЮТСЯ (полноценность приложения); переделываем центр и правую панель.

## Зачем
Сейчас центр — только чат, а дифф/редактор теснятся в правой панели. Планы, которые агент пишет (фазы, разборы), вообще не видны — только сырым текстом в чате. Ты хочешь: **центр = вкладки** (тред, план-markdown, дифф, файл — как в Cursor), **правая панель = навигатор** (ссылки, а не контент), **агент ссылается на файлы планов, а не вываливает их в чат**.

## Модель (простым языком)

### Центр — вкладки (как VSCode/Cursor)
Полоса вкладок в шапке карточки. Типы вкладок:
- **Тред** 💬 — чат с агентом. Тредов у задачи может быть несколько (ядро уже умеет): вкладка на тред, «+» — новый тред, история старых тредов открывается из навигатора.
- **Markdown** 📄 — просмотр .md красивым рендером (план, фазы, CLAUDE.md, PROGRESS.md). Открывается по клику на ссылку в чате или из навигатора. Кнопка «редактировать» переключает в редактор.
- **Дифф** ± — изменения задачи (все репо или один), с комментариями пачкой как сейчас.
- **Файл** ✎ — редактор CodeMirror (⌘S, ⌘L как сейчас).

Правила:
- Композер виден только в тред-вкладках.
- Агент в чате пишет «план в `docs/plan/redesign.md`» — **ссылка кликабельна** → открывается md-вкладка. В чате плана НЕТ.
- Вкладки задачи запоминаются (перезапуск приложения не теряет).
- ⌘W закрыть вкладку, ⌘1..9 — переключение? (вопрос ниже — конфликт с задачами)

### Правая панель — навигатор задачи (секции, только ссылки)
1. **Цель** — одна строка + ссылка «открыть» → PROGRESS.md вкладкой (НЕ рендер в панели).
2. **Изменения** — репо + ✎N +A −D → клик открывает дифф-вкладку.
3. **Файлы проекта** — корневой CLAUDE.md, CLAUDE.md каждого репо, PRODUCT.md, docs/plan/* → md-вкладки. (Твоё «легко видеть планы и CLAUDE.md проекта».)
4. **Вложения** — всё, что ты прикреплял ⌘L по этой задаче (файл:строки) → клик открывает ТОТ тред на ТОМ сообщении (скролл+подсветка).

### «Показать файлы» проекта
Дерево остаётся в сайдбаре, но файл открывается **вкладкой по центру** (не в правой панели). Правая панель при этом не занята.

## Решения (2026-07-05)
1. ⌘1-9 по задачам УБИРАЕМ (Газик не пользуется). Вкладки: **⌘⇧[ / ⌘⇧]** — влево/вправо, ⌘W — закрыть.

3. «Кусок плана → в чат» — ОТЛОЖЕНО (Газик пока не понял нужность).
4. **Проектный режим (Show files v2)** — НОВОЕ решение Газика: глобальный Show files остаётся и становится режимом работы с проектом напрямую:
   - открытие проектного Show files → **общий глобальный чат проекта** (агент работает в ОСНОВНОЙ рабочей копии, изменения локальные, не в worktrees);
   - открываемые файлы — те же вкладки по центру;
   - правая панель проектного режима: изменения рабочей копии, ветка каждого репо, глобальные файлы планов.
   - Тех-вопросы (решить при реализации): треды уровня проекта в state (schema v4: thread без task_id), guardrails для агента в main-копии (не мусорить .claude в репо юзера — вероятно, флаг --settings).
