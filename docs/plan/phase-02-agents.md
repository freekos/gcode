# Фаза 2 — Агенты и threads 📖

## Зачем
Ядро умеет готовить рабочее место (задача + worktrees). Теперь в это место надо посадить **агента**: запустить Claude Code внутри задачи, видеть его работу вживую, продолжать разговор, и **технически** гарантировать, что git ему недоступен. После этой фазы gcode перестаёт быть «менеджером папок» и становится agentic-инструментом.

## Что получишь в конце (пощупать)
```bash
gcode thread new azi fix-login "почини редирект после логина"
# → в корне задачи стартует Claude Code, ответ стримится прямо в терминал,
#   тред записан; агент видит все worktree задачи, но git ему запрещён

gcode thread continue azi fix-login "теперь добавь тест"   # продолжение того же разговора
gcode thread ls azi fix-login                              # треды задачи с их статусами
gcode task ls azi   # статусы задач теперь живые: running / review — из фактов, не из слов
```

## Как будем делать (простым языком)

### 1. Запуск агента — headless Claude Code
Агент = обычный процесс `claude`, запущенный **в корне задачи** (там, где TASK.md и worktrees-подпапки):
- `claude -p "<промпт>" --output-format stream-json` — Claude отдаёт события машинным потоком (**stream-json**: каждая строка — JSON-событие «кусочек текста», «вызвал инструмент», «закончил»);
- мы читаем поток и показываем текст по мере генерации;
- корень задачи как cwd — ключ к «знанию»: Claude Code сам подхватывает CLAUDE.md/контекст из текущей папки и вложенных репо.
Уроки из cc уже учтены: флаг `--include-partial-messages` (иначе поток без дельт), фолбэк на целые сообщения, stderr — в честную ошибку, а не в тишину.

### 2. Threads — метаданные у нас, транскрипт у Claude
**Тред** = один разговор с агентом внутри задачи. Схема хранения:
- **Claude Code сам хранит транскрипт** (история привязана к папке задачи — то самое cwd-правило, проверенное на переносе твоих чатов);
- **мы храним метаданные** в SQLite: тред → задача, session-id (чтобы делать resume), движок, заголовок, время;
- `thread continue` = `claude --resume <session-id>` из того же корня. Никакой своей истории мы не изобретаем — переиспользуем нативную, поэтому те же чаты открываются и обычным `claude -c` из терминала.

### 3. Git запрещён агенту — технически, не просьбой
При создании задачи (расширяем provisioning фазы 1) в корень пишутся:
- `.claude/settings.json` с **deny-правилом на git** (и на другие опасные команды) — Claude Code не даст агенту выполнить `git …`, это не зависит от его «послушности»;
- `CLAUDE.md` с правилами задачи («ты работаешь только в этих папках, git делает человек»).
Плюс тест: агент-имитация пытается `git commit` → получает отказ.

### 4. Статусы задач — из фактов
- **running** — процесс агента жив (по pid);
- **review** — агент завершился (exit-код есть, работа ждёт твоего ревью);
- **needs_input** — агент задал вопрос и остановился (маркер в выводе; полноценно — в UI-фазе).
Статус меняет ядро по фактам процесса — не по фразам агента.

### 5. Движки — абстракция сразу
Внутри — интерфейс «движок» (запустить / продолжить / парсить поток). В этой фазе реализация одна — Claude Code; Codex добавится позже без перестройки (интерфейс уже стоит).

### 6. Один агент на задачу одновременно
Worktree один — два агента, пишущих в одни файлы параллельно, устроят кашу. Правило ядра: **на задачу — один запущенный агент одновременно** (второй `thread new/continue` в тот же момент вежливо откажет «агент уже работает»). Тредов-разговоров при этом может быть сколько угодно — последовательно.

## Риски и вопросы к тебе
1. **Режим прав агента:** предлагаю `--permission-mode bypassPermissions` + жёсткий deny-список (git, push-команды, rm вне worktree) — максимум автономии в коде, ноль доступа к git (как было в cc). Ок?
2. **Тестовый агент:** e2e-тесты гоняются на **фейковом** `claude` (скрипт, отвечающий готовым stream-json) — быстро и бесплатно; настоящий Claude Code проверяем одним живым смоуком в конце. Ок?
3. **Интерактивность:** в этой фазе чат = «отправил сообщение → дождался ответа» (one-shot + continue). Полноценный живой чат с вводом по ходу — это UI фазы 3. Ок, или нужен интерактив уже в терминале?
