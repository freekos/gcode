# Фаза 4 — Editor + Diff review 🔨

## Зачем
Сейчас видно ЧТО агент трогал (✎3, +49 −2), но не видно САМИ изменения — приходится открывать другой редактор. Эта фаза замыкает главную петлю ревью: **посмотрел дифф → оставил комментарий к строкам → агент получил и поправил** — не выходя из gcode. Плюс лёгкая правка файлов руками, когда проще поправить самому, чем объяснять.

## Что получишь в конце (пощупать)
- Клик по репо в правой панели (или вкладка «Изменения» в карточке) → **дифф рабочего дерева** этого репо: файлы слева, изменения справа, красное/зелёное.
- **Выделил строки в диффе → «Комментарий агенту»** → пишешь «тут утечка, почини» → в тред уходит сообщение с точным контекстом (файл:строки + код) → агент чинит. Это паттерн Orca/cc, наша главная фича ревью.
- Открытие любого файла worktree в **редакторе с подсветкой** (правка + ⌘S сохранить).
- Всё в нашем дизайне: тёмная тема на наших токенах.

## Как будем делать (простым языком)

### 1. Движок редактора — CodeMirror 6 (решение из ресёрча 3a)
Лёгкий (~300KB против 2-5MB у Monaco), стартует мгновенно, из него собраны редакторы Sourcegraph/Replit. Официальный модуль **@codemirror/merge** даёт side-by-side дифф из коробки. Тему соберём из наших токенов — впишется идеально.

### 2. Дифф — из фактов git (ядро)
Новая команда ядра: полный `git diff` worktree по репо (staged+unstaged+новые файлы), разобранный на файлы и куски (hunks). UI просто рисует. Тесты — на настоящих временных репо, как всегда.

### 3. Комментарии к строкам → агенту
Выделение строк в дифф-вью → поле комментария → формируется сообщение в тред:
«В `server/src/auth.ts` строки 40-44: [код] — Комментарий: тут утечка». Агент получает точный контекст и работает. Несколько комментариев можно накопить и отправить пачкой (как ревью в GitHub).

### 4. Где это живёт (UX)
В карточке появляются **вкладки: «Тред» | «Изменения»** (паттерн Conductor). «Изменения» = файлы с диффстатами слева, дифф справа. Клик по ✎ в правой панели открывает эту вкладку сразу на нужном репо. Клик по файлу в тул-строке треда (`Edit · auth.ts`) — тоже.

## Решения-развилки (мои рекомендации)
1. **Порядок:** сначала дифф+комментарии (петля ревью), правка файлов — вторым шагом фазы. Ревью нужнее ежедневно.
2. **Дифф-вью:** unified (одна колонка, как GitHub) по умолчанию — компактнее на ноутбуке; side-by-side тумблером.
3. **Файлового дерева пока нет** (как у Orca) — вход через дифф/тул-строки/⌘P покрывает 95% случаев, дерево добавим по запросу.

## Журнал правок (по ревью)
- Тред по ZCode-референсу: контент и композер в колонке 860px по центру, юзер-пузыри с кромкой (max 66%), разделители ходов «Работал Xс ——» с замером длительности, inline-`код` чипами в тексте агента, композер оторван от краёв (радиус 16).

- Комментирование переведено на GitHub-паттерн (по фидбеку «непонятно, почему синим»): строки — обычный текст (свободно копируются), синий «+» в gutter на hover — явный якорь действия, shift+клик по «+» — диапазон; выделение — слитный мягкий фон с одной левой полосой вместо рамки на каждой строке.

- По второй итерации фидбека: клик для комментария — по ВСЕЙ строке (плюс в gutter остался как подсказка), дифф переехал из вкладок в ПРАВУЮ панель (расширяется до 44%, тред остаётся на виду; × или ⌘D закрывает); починены некликабельные кнопки шапки (невидимая drag-полоса перекрывала шапку — теперь drag-зоной служит сама шапка треда, кнопки в ней работают).

- UX-заход 3: правая панель resizable (обычная 210-420, дифф-режим 460-60% экрана, запоминается); базовый текст 13.5px + зум ⌘+/⌘−/⌘0 (0.7-1.5, персистентный); Esc снимает выделение строк в диффе; все textarea авто-растут до 220px и дальше скроллят (паттерн Linear/Slack).
